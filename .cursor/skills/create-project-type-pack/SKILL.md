# Skill: Create Project Type Pack

## 使用场景

当系统需要支持一种新的业务项目类型时使用。
当前支持的项目类型：自媒体内容运营 / 账号矩阵管理 / 网站运营与SEO / 线索获客。

新增类型不修改已有类型逻辑，只新增配置包。

---

## 项目类型配置包包含

### 1. 类型定义（在 project schema 中新增枚举值）
```typescript
// src/modules/tenant/schemas/project.ts
// 新增 ProjectType 枚举值，不修改已有值
```

### 2. 目标类型配置
该项目类型支持哪些目标（Goal）类型：
- 目标名称（中文）
- 目标的度量方式（数量、百分比、是/否）
- 典型指标示例

### 3. 交付标准定义（Deliverable）
该项目类型的标准交付物：
- 交付物名称
- 验收标准描述
- 典型示例

### 4. 推荐流程模板 ID
该项目类型推荐关联哪些 WorkflowTemplate（system 预置模板）。

### 5. 推荐 Agent 角色组合
该项目类型的典型 Agent 团队配置：

| Agent 角色 | 职责 | 对应 AgentTemplate |
|---|---|---|
| （按具体类型定义） | | |

### 6. Mock 数据更新
在 `projectTypeMock.ts` 中新增该类型的 mock 配置。

## 四种当前类型参考

### 自媒体内容运营
- 目标：发布内容数、涨粉数、互动率
- 交付物：已发布帖子、数据报告
- 终端：社交媒体账号（TikTok / Instagram / 小红书等）

### 账号矩阵管理
- 目标：矩阵账号覆盖率、内容差异度
- 交付物：矩阵发布记录、各账号数据汇总
- 终端：同平台多账号（每账号绑定独立 Identity）

### 网站运营与 SEO
- 目标：关键词排名、文章发布数、自然流量增长
- 交付物：已发布文章、排名变化报告
- 终端：CMS / WordPress / 网站 API

### 线索获客
- 目标：新增私域联系人数（WhatsApp / Line 加粉量）
- 交付物：私域联系人数增量、引流内容发布记录
- 终端：WhatsApp Business、Line 官方账号、落地页

---

## 禁止行为

- 禁止修改已有项目类型的枚举值或逻辑
- 禁止把新类型的专有逻辑硬编码进通用页面组件
- 禁止跳过 mock 数据直接上页面
