# AI Work Control Center - 项目创建流程 1.0

## 1. 文档目的

本文档用于定义系统中"项目创建流程 1.0"的产品目标、页面结构、步骤设计和输出结果。

项目创建流程不是普通的新建表单，而是把一个真实业务项目初始化为系统中的可执行对象。

---

## 2. 当前阶段目标

当前阶段要实现：

**通过多步骤向导，创建一个具备业务约束和执行基础的真实项目。**

项目创建后，应至少包含：

- 项目基础信息
- Goal
- Deliverable
- Resources
- Identity
- Terminal
- SOP
- 推荐流程占位

---

## 3. 为什么要做项目创建流程

当前系统已经具备：

- 项目模型
- Identity
- Workflow / Task 基础
- 项目工作台

下一步需要让用户能"从 0 到 1 正确创建项目"。

否则：

- Project 仍然只是管理对象
- Goal / Deliverable / SOP 只能后补
- 流程和任务缺少真实业务起点

---

## 4. 建议采用的交互方式

当前阶段推荐使用：

**多步骤向导**

不建议使用：

- 单页长表单
- 仅弹窗创建
- 仅输入项目名称后再手工补齐全部配置

---

## 5. 建议步骤

### Step 1：项目基础信息
包含：
- 项目名称
- 项目类型
- 项目描述
- 负责人

---

### Step 2：目标 Goal
包含：
- 目标类型
- 目标名称
- 目标描述
- 成功标准
- KPI 说明

Goal 由人工确认，不允许漂移。

---

### Step 3：交付 Deliverable
包含：
- 交付类型
- 交付内容
- 频率
- 目标值
- 单位

Deliverable 用于定义项目的可验收输出。

---

### Step 4：资源与身份
包含：
- 绑定 Identity
- 设置默认 Identity
- 绑定 Terminal（当前可轻量化）
- 资源摘要

---

### Step 5：SOP
包含：
- 自然语言 SOP 输入
- SOP 说明
- 解析结果占位（当前阶段只做占位）

---

### Step 6：推荐流程
根据：
- Goal
- Deliverable
- SOP
- Identity

给出"推荐流程模板"占位或建议，不做复杂智能解析。

---

### Step 7：确认创建
展示项目创建摘要，包括：

- 项目基础信息
- Goal
- Deliverable
- Identity
- Terminal
- SOP
- 推荐流程

用户确认后创建项目。

---

## 6. 项目创建后的结果

项目创建成功后，系统至少应完成：

- 创建 Project
- 创建 ProjectGoal
- 创建 ProjectDeliverable
- 写入 ProjectResourceConfig
- 写入 ProjectSOP
- 建立 Identity 绑定关系
- 建立 Terminal 绑定关系（当前可轻占位）
- 保存推荐流程占位信息

---

## 7. 页面与结构要求

项目创建流程必须：

- 符合统一后台 UI 风格
- 步骤清晰
- 支持返回上一步
- 支持最终摘要确认
- 不让用户迷失在复杂字段中

---

## 8. 当前阶段边界

当前阶段不做：

- 复杂 LLM 推荐
- 自动生成完整 Workflow
- 自动创建 Task
- 批量项目导入
- 第三方账号真实授权流程

---

## 9. 当前阶段成功标准

项目创建流程 1.0 完成后，应满足：

1. 用户可以从系统中真正创建一个结构完整的项目
2. 项目创建不再是空壳
3. Goal / Deliverable / Identity / SOP 能在创建时被明确下来
4. 项目创建后可直接进入项目详情工作台继续使用
