# Phase 10 冻结基线

本文档锁定 Phase 10（Workflow Planning Session 1.0）验收完成后形成的基线。后续阶段应遵守以下约束。

---

## 1. 冻结规划域四大对象

以下对象从现在开始视为规划域正式基线：

- **WorkflowPlanningSession**
- **WorkflowPlanningDraft**
- **WorkflowPlanningDraftNode**
- **WorkflowPlanningMessage**

第十一阶段可以扩展字段，但不要改掉它们的基本职责。

---

## 2. 冻结 Session / Draft / Template / Instance 四层边界

必须保持：

| 层 | 定位 |
|----|------|
| **Session** | 规划会话 |
| **Draft** | 草案版本 |
| **Template** | 标准流程模板 |
| **Instance** | 运行实例 |

后面任何阶段都不能混用。

---

## 3. 冻结三栏式工作台结构

规划工作台固定保持：

| 栏位 | 内容 |
|------|------|
| **左栏** | 规划上下文 |
| **中栏** | 消息会话 |
| **右栏** | 当前草案 |

第十一阶段可以增强，但不要改成普通聊天页。

---

## 4. 冻结 DraftNode 与 TemplateNode 的兼容方向

DraftNode 允许继续增强，但整体要保持向 `WorkflowTemplateNode` 兼容，方便第十二阶段做 Draft → Template 转换。

---

## 5. 冻结中文显示约束

从第十阶段起，所有后续模块必须默认遵守：

- 前台优先中文
- 枚举统一中文映射
- 标签集中维护
- 错误提示与空状态中文化

这条不再是建议，而应视为默认规则。

---

## 6. 冻结 mock 交互契约

当前规划域 service / repository 的命名和职责建议锁住，不要在第十一阶段大改接口名。

---

## 7. 冻结「第十阶段不做项」

第十一阶段之前，仍然不要把这些强行塞回来：

- 模板发布确认
- 真实运行态接入
- 真实执行监督
- 复杂差异比对器
- OpenClaw 主路径

---

## 8. 冻结规划会话与前台中文说明的语义

不要把「流程规划会话」页面重新命名成模糊概念，例如「聊天助手」或「AI 对话页」。它必须保持流程规划语义。
