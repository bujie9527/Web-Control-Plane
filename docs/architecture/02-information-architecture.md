# AI Work Control Center - 信息架构

## 1. 文档目的

本文档用于定义系统后台的信息架构、菜单结构、页面关系和业务路径。

重点不是技术实现，而是帮助产品设计、架构设计和前端实现保持一致。

---

## 2. 顶层信息架构

系统分为两个控制台：

- 平台后台（Platform Console）
- 租户后台（Tenant Console）

---

## 3. 平台后台信息架构

- 平台工作台

### 3.7 平台设置

## 5. 租户后台页面职责

### 5.1 工作台
登录后的默认首页，展示：

- 核心指标卡
- 我的待办
- 预警中心
- 最近任务
- 快捷入口

它的作用不是单纯看数据，而是给租户用户提供“今日工作入口”。

---

### 5.2 项目中心
业务中心。

负责管理：

- 项目列表
- 项目模板
- 项目详情
- 项目归档

项目详情页是系统最重要的业务页面。

---

### 5.3 任务执行
执行中心。

负责：

- 任务中心
- 运行中任务
- 待审核任务
- 异常任务
- 已完成任务

---

### 5.4 流程中心
标准化中心。

负责：

- 流程列表
- 流程设计器（第一阶段先做骨架）
- 流程模板
- 节点模板
- 流程版本

---

### 5.5 Agent中心
AI 团队组织中心。

负责：

- Agent 库
- Agent 团队
- 角色模板
- Prompt 管理
- 模型分配

---

### 5.6 Skills能力库
复用能力中心。

负责：

- 技能总览
- 技能分类
- 技能测试台
- 技能版本

---

### 5.7 终端中心
执行入口管理中心。

负责：

- 终端总览
- 社媒账号
- Web自动化
- 系统终端
- API接入
- 执行日志

---

### 5.8 数据分析
结果闭环中心。

负责：

- 数据总览
- 项目分析
- 渠道分析
- 任务分析
- Agent分析
- Skill分析
- KPI中心

---

### 5.9 系统管理
租户级后台管理中心。

负责：

- 成员管理
- 角色权限
- 菜单权限
- 工作区设置
- 审计日志

---

## 6. 项目详情页信息架构

项目详情页使用页内二级导航，建议结构如下：

- 概览
- 目标与KPI
- 渠道配置
- Agent团队
- 终端分配
- 流程与任务
- 结果反馈
- 项目设置

这是一个“工作台型详情页”，不是普通对象详情页。

---

## 7. 业务路径设计

### 路径一：从业务出发
项目中心 → 项目详情 → 渠道 / Agent / 终端 → 流程与任务 → 结果反馈

### 路径二：从能力出发
Skills能力库 / Agent中心 / 流程中心 → 项目中心启用 → 任务执行

### 路径三：从执行出发
终端中心 → 任务执行 → 异常处理 → 数据分析

### 路径四：从优化出发
数据分析 → 流程中心 / Agent中心 / Skills能力库 → 再回到执行

---

## 8. 第一阶段建议

第一阶段虽然会展示很多模块，但重点应优先打磨：

1. 双后台主框架
2. 平台租户管理页
3. 租户工作台
4. 项目列表页
5. 项目详情工作台页
6. 通用列表页和详情页模式

这些页面会决定整个后台的产品感。
