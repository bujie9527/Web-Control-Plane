# AI Work Control Center - 第一阶段开发路线图

## 1. 文档目的

本文档用于定义 AI Work Control Center 第一阶段的开发推进顺序、模块优先级、阶段目标、验收标准和执行原则。

它服务于以下场景：

- 产品推进排期
- Cursor 分阶段开发
- 团队协作分工
- 阶段性验收
- 架构稳定性控制

本路线图仅覆盖第一阶段：**企业级中控后台控制平面骨架搭建**。

---

## 2. 第一阶段总目标

第一阶段的目标不是完成全部业务能力，而是先完成一个可商用、可演示、可扩展的后台控制平面骨架。

必须达到的效果：

1. 有完整的登录入口和身份结构
2. 有清晰的双后台体系
3. 有多租户基础结构
4. 有统一的后台 UI 风格和布局
5. 有清晰的菜单体系和业务路径
6. 有标准化页面骨架
7. 有模块化目录结构
8. 有 mock-first 的数据与接口规范
9. 有后续 Agent / Skill / Terminal / Workflow 可插拔扩展位

## 4. 开发总体原则

### 原则 1：先骨架，后逻辑
优先完成后台壳、结构、布局、菜单、模块边界，再逐步接业务。

### 原则 2：先平台基础，再业务核心
先把平台后台与租户后台的基础控制面搭出来，再进入项目和流程等业务页面。

### 原则 3：优先打磨“关键演示页”
不是所有页面都要同等深度，第一阶段应重点打磨：
- 登录页
- 平台租户管理页
- 租户工作台
- 项目列表页
- 项目详情工作台页

### 原则 4：mock-first
所有页面先使用 mock 数据和 mock service 驱动。

### 原则 5：结构统一
列表页、工作台页、设置页、分析页要分别形成稳定模板，不要每页都重新发明结构。

### 原则 6：复杂任务分阶段
不一次性让 Cursor 完成全部任务，要按阶段拆分执行并逐步验收。

---

## 5. 阶段划分

第一阶段建议拆成 6 个开发轮次：

- Phase 0：项目基础与规范建立
- Phase 1：认证、布局与双后台骨架
- Phase 2：平台后台核心页面
- Phase 3：租户后台核心页面
- Phase 4：项目详情工作台与关键业务页
- Phase 5：通用基础设施补齐与结构清理

---

## 6. Phase 0：项目基础与规范建立

### 6.1 目标
建立整个项目的开发基础，不做页面业务，先把工程规则、目录结构、文档和 Cursor 基础能力固定下来。

### 6.2 本阶段内容
- 初始化项目目录
- 建立 `.cursor/rules`
- 建立 `.cursor/skills`
- 建立 `docs/architecture`
- 建立 `app / components / modules / core` 基础结构
- 建立导航、权限、request/response、schema、event、audit 的基础目录
- 明确平台后台和租户后台的路由空间

### 6.3 产出物
- 项目基础目录
- Rule 文件
- Skill 文件
- 架构文档
- 基础规范文档

### 6.4 验收标准
- 项目结构可运行
- 文档齐全
- 双后台路由规划明确
- Cursor 已具备基础工作规范

### 6.5 优先级
P0

## 10. Phase 4：项目详情工作台与关键业务页

### 10.3 设计要求
项目详情页必须是：
**工作台型详情页**

不是：
- 普通对象详情页
- 长表单页
- 单页信息堆叠页

### 10.4 页面深度要求
这是第一阶段最高优先级打磨页面，所有子模块都应做到“骨架真实、结构完整、可演示”。

- 页内导航清晰

## 13. 第一阶段的关键演示路线

为了产品展示和开发验收，建议把第一阶段演示路径固定为：

### 路线一：平台视角
登录 → 平台后台 → 租户列表 → 租户详情 → 资源与配额

### 路线二：租户视角
登录 → 租户后台工作台 → 项目列表 → 项目详情工作台

### 路线三：能力视角
项目详情 → Agent团队 / 终端分配 / 流程与任务 → 结果反馈

只要这三条路线打通，第一阶段就已经具备很强的展示力。

## 14. 每轮开发的执行方式建议

每一轮都建议按以下方式驱动 Cursor：

1. 先指定当前 Phase
2. 让 Cursor 阅读相关架构文档
3. 先输出实施计划
4. 你确认后再执行
5. 执行后做结构自检

推荐使用模式：

### 第一步：先计划
说明：
- 本轮目标
- 涉及页面
- 涉及模块
- 需新增哪些组件
- 风险点

### 第二步：再实现
逐步生成代码，不一次覆盖过多文件。

### 第三步：最后自检
检查：
- 双后台分离
- UI 统一
- 命名统一
- 是否误改无关文件
- 是否保留扩展接口

## 15. 阶段验收总表

### 第一阶段完成时，应满足：

#### 系统层
- 有双后台
- 有权限骨架
- 有多租户基础模型
- 有统一 Layout 与菜单结构

#### 页面层
- 有完整登录入口
- 有平台租户管理页
- 有租户工作台
- 有项目列表页
- 有项目详情工作台页
- 其他核心模块至少有高质量骨架

#### 工程层
- 有清晰目录结构
- 有 rules / skills / docs
- 有统一 schema / service / repository / mock 规范
- 有 API contract 基础约束
- 有 event / audit 占位

#### 观感层
- 页面有企业级后台感
- 侧边栏风格稳定
- 页面结构一致
- 可以直接用于对外演示或内部评审

## 16. 第二阶段的衔接方向

第一阶段完成后，第二阶段建议沿以下顺序推进：

1. 真实权限体系
2. 真实数据库与数据持久化
3. Workflow 逻辑增强
4. Agent Team 配置增强
5. Skill 测试台能力增强
6. Terminal 接入层
7. 任务执行状态机
8. 结果分析回流
